## Problem statement (one paragraph)

Architects need to explore many floor-plan options that satisfy functional, regulatory, and connectivity constraints. Current generative tools output images, not structured BIM data. This project aims to translate natural-language requirements into **vector / graph representations** and export **IFC** files that professionals can open, edit, and refine in Vectorworks (or Revit, Archicad, etc.).

## Architecture direction (hypothesis — not decided)

```
Text (PoR)  →  LLM / parser  →  Constraint spec (JSON)
                                      ↓
                              Layout generator (graph + geometry)
                                      ↓
                              Constraint solver (validation / repair)
                                      ↓
                              IFC writer  →  .ifc file
```

Preferred output path: **IFC first**, not vendor-specific `.vwx`.

## Conventions for agents

### Code
- Python **≥ 3.11**, package lives under `src/vector_floorplan_ai/`.
- Match existing style; run `make lint` before finishing.
- **Minimize scope** — small, focused changes aligned with the current roadmap item.
- Prefer **IFC** interoperability over Vectorworks-specific APIs unless the user says otherwise.

### Tests
- Default to **short unit tests** in `tests/`.
- One logical assertion per behavior where possible.
- Mark slow or data-heavy tests with `@pytest.mark.slow` or `@pytest.mark.integration`.
- Do not add tests that only assert imports or trivial constants unless scaffolding.

### Data
- Never commit large binaries; use `datasets/` with sidecar metadata.
- Synthetic data generation is in scope; scraping copyrighted plans is out of scope unless explicitly approved.

### Documentation
- **Only update `AGENTS.md` when the user asks** — e.g. "update AGENTS.md with our decision on X".
- Update `README.md` for user-facing changes; keep `AGENTS.md` as the agent-oriented project brain.

### Git

## Domain glossary

| Term | Meaning |
|------|---------|
| **PoR** | Program of Requirements — textual list of spaces, areas, adjacencies, rules |
| **IFC** | Industry Foundation Classes — open BIM interchange standard |
| **Spatial graph** | Nodes = rooms/zones; edges = doors, openings, required adjacency |
| **Constraint satisfaction** | Hard rules (min area, connectivity, daylight) enforced by solver or repair loop |
